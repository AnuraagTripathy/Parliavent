# Parliavent product rules

## Core idea

Parliavent is a debate platform where users write arguments before posting. Before an argument goes live, it is reviewed by an AI judge that highlights weak points in the argument.

The judge can flag:
- factual claims that need evidence
- unsupported, vague, or overbroad information
- possible logical fallacies
- unclear or emotionally loaded wording

The product should feel like a debate coach, not a gatekeeper.

## Non-negotiable rule

The AI must never silently rewrite or post the user's argument.

Every change to the user's argument must come from an explicit user action:
- Use this wording
- Replace phrase
- Attach source
- Mark as opinion
- Keep as-is
- Dispute
- Edit first

The published argument must always be the user's words, optionally including suggestions they explicitly accepted.

## What the judge does

The judge should:
1. Fact-check stated facts.
2. Flag unsupported, vague, or overbroad information.
3. Identify possible logical fallacies.
4. Name the logical fallacy.
5. Explain the fallacy in plain English.
6. Give a simple example so the user understands it.
7. Suggest clearer wording to reduce miscommunication.

## What the judge must not do

The judge must not:
- Write a final approved argument.
- Replace the user's voice with generic polished language.
- Shame the user.
- Present fallacy detection as certain.
- Block casual posting by default.
- Apply edits automatically.

## Tone

The judge should sound calm, helpful, and precise.

Good copy:
- "This may be too broad."
- "This needs evidence."
- "This could be read as a false dilemma."
- "Readers may dismiss this as exaggeration."

Bad copy:
- "Your argument is wrong."
- "This is invalid."
- "You committed a fallacy."
- "This is misinformation."
